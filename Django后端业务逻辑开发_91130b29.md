## 三、数据库物理实现（Models）

> 策略：用 Django ORM 定义模型，`makemigrations + migrate` 自动建表，保持与你讲稿中 SQL 结构完全一致。

### 3.1 users/models.py — 用户模型

核心设计：**自定义 User 模型继承 AbstractUser**，用 `user_type` 字段区分学生/教师，而不是分两张表。好处是 Django 内置认证系统无缝兼容。

```python
from django.contrib.auth.models import AbstractUser
from django.db import models

class User(AbstractUser):
    USER_TYPE_CHOICES = (
        ('student', 'Student'),
        ('teacher', 'Teacher'),
    )
    PERMISSION_CHOICES = (
        ('admin', 'Admin'),
        ('teacher', 'Teacher'),
    )

    user_type = models.CharField(max_length=10, choices=USER_TYPE_CHOICES)
    # 以下为教师专属字段（学生时为空即可）
    permission_level = models.CharField(
        max_length=10, choices=PERMISSION_CHOICES,
        null=True, blank=True
    )
    # 学生专属：关联归属教师
    teacher = models.ForeignKey(
        'self', on_delete=models.SET_NULL,
        null=True, blank=True,
        related_name='students',
        limit_choices_to={'user_type': 'teacher'}
    )

    class Meta:
        db_table = 'user'
```

> 关键决策：使用单表继承而非讲稿中的 Student/Teacher 双表。Django 的认证系统要求一个 `AUTH_USER_MODEL`，单表方案代码更简洁，且 `user_type` 字段足以区分角色，`teacher` 自引用外键实现 "教师管理学生" 的 1:n 关系。

from rest_framework.permissions import BasePermission

class IsTeacher(BasePermission):
    def has_permission(self, request, view):
        return request.user.is_authenticated and request.user.user_type == 'teacher'

class IsAdmin(BasePermission):
    def has_permission(self, request, view):
        return request.user.is_authenticated and request.user.permission_level == 'admin'

class IsOwnerOrTeacher(BasePermission):
    """只有本人或教师可以访问"""
    def has_object_permission(self, request, view, obj):
        return request.user == obj or request.user.user_type == 'teacher'

### 5.3 关键业务逻辑：创建题目时同时创建答案和测试用例

```python
class QuestionViewSet(ModelViewSet):
    def perform_create(self, serializer):
        # 1. 保存题目
        question = serializer.save(teacher=self.request.user)
        # 2. 批量创建答案
        answers_data = self.request.data.get('answers', [])
        for ans in answers_data:
            Answer.objects.create(question=question, correct_sql=ans['correct_sql'])
        # 3. 批量创建测试用例
        test_cases_data = self.request.data.get('test_cases', [])
        for tc in test_cases_data:
            TestCase.objects.create(question=question, **tc)
```

## 七、模块四：提交与判题 API

### 7.1 判题服务对接（apps/submissions/judge.py）

这是你与队友的**接口协议**。你的后端收到学生提交的 SQL 后，调用队友开发的 Docker 判题服务，拿到结果后存入数据库。

```python
import requests
from django.conf import settings

def judge_submission(submitted_sql, test_cases, create_table_sql=''):
    """
    调用判题模块（队友提供的 Docker 服务）
    
    参数:
        submitted_sql: 学生提交的 SQL 语句
        test_cases: [{'test_input': '...', 'expected_output': '...'}, ...]
        create_table_sql: 建表语句（由题目提供）
    
    返回:
        {
            'passed': True/False,
            'execution_status': 'ACCEPTED|WRONG_ANSWER|TIMEOUT|ERROR',
            'score': 100,
            'details': [{'test_case_id': 1, 'passed': True, 'actual_output': '...'}, ...]
        }
    """
    response = requests.post(
        f'{settings.JUDGE_SERVICE_URL}/api/judge/',
        json={
            'sql': submitted_sql,
            'create_table_sql': create_table_sql,
            'test_cases': test_cases,
        },
        timeout=30  # 30秒超时
    )
    return response.json()
```

> **重要**：你需要和队友提前约定好这个 API 契约（URL、请求体格式、响应体格式），在 `settings.py` 中配置 `JUDGE_SERVICE_URL`。

### 7.2 提交视图（apps/submissions/views.py）

| 端点 | 方法 | 功能 | 权限 |
|------|------|------|------|
| `/api/submissions/` | POST | 提交 SQL（触发判题） | 学生 |
| `/api/submissions/` | GET | 查看提交历史（支持按题目/考试筛选） | 已登录 |
| `/api/submissions/{id}/` | GET | 查看单次提交详情 | 已登录 |

```python
class SubmissionViewSet(ModelViewSet):
    
    @action(detail=False, methods=['post'])
    def submit(self, request):
        question_id = request.data['question_id']
        exam_id = request.data.get('exam_id')
        submitted_sql = request.data['submitted_sql']
        
        # 校验：考试模式下需检查时间
        if exam_id:
            exam = Exam.objects.get(id=exam_id)
            now = timezone.now()
            if not (exam.start_time <= now <= exam.end_time):
                return Response({'error': '不在考试时间内'}, status=400)
        
        # 获取题目和测试用例
        question = Question.objects.get(id=question_id)
        test_cases = list(question.test_cases.values('test_input', 'expected_output'))
        
        # 调用判题服务
        result = judge_submission(submitted_sql, test_cases, question.create_table_sql)
        
        # 保存提交记录
        submission = Submission.objects.create(
            student=request.user,
            question=question,
            exam_id=exam_id,
            submitted_sql=submitted_sql,
            execution_status=result['execution_status'],
            score=result['score'],
        )
        return Response(SubmissionSerializer(submission).data)
```

### 8.1 成绩统计实现（Django ORM 聚合）

```python
from django.db.models import Count, Q, Avg, Max

# 题目完成率
def question_completion_rate(question_id):
    total = Submission.objects.filter(question_id=question_id).values('student').distinct().count()
    passed = Submission.objects.filter(
        question_id=question_id, execution_status='ACCEPTED'
    ).values('student').distinct().count()
    return {'total_attempted': total, 'passed': passed, 'rate': passed / total if total else 0}

# 考试排名
def exam_ranking(exam_id):
    # 取每个学生在本次考试中每题的最高分，汇总排名
    submissions = Submission.objects.filter(exam_id=exam_id).values('student__username').annotate(
        total_score=Max('score')
    ).order_by('-total_score')
    return list(submissions)
```
